# Commenting Guidelines

These conventions describe how to annotate code across the project.

## Path Banner

Every file should start with a banner comment showing its location relative to the `src` directory.

```ts
// src/<path>
```

## Hooks

Custom hooks must include a short summary block at the top describing their purpose and behavior.

## Inline Comments

Use inline comments to explain any non-obvious logic so future maintainers understand the intent.

## JSX Markup

Large or complex markup sections should be labeled with JSX comments to clarify structure.

## Design Notes

When making unusual or non-obvious design decisions, briefly note the reasoning in a comment.
